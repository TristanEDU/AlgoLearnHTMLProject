# Tristan Thomas Portfolio Website

## Project Overview
This is a complete multi-page portfolio website built entirely with HTML and CSS, showcasing my skills as a web developer. The site demonstrates mastery of semantic HTML, accessibility best practices, and modern web development standards.

## Project Structure
```
project-root/
├── index.html          # Homepage with hero section and featured projects
├── about.html          # About page with biography and journey
├── projects.html       # Portfolio projects showcase
├── contact.html        # Contact form and information
├── css/
│   ├── style.css       # Main stylesheet
│   └── components.css  # Component-specific styles
└── images/             # Project images and assets
```

## Pages Included

### 1. Homepage (index.html)
- Hero section with profile image and introduction
- Featured projects showcase with three project cards
- Skills overview section
- Call-to-action buttons for viewing work and contact

### 2. About Page (about.html)
- Personal biography and background
- Journey into web development
- Development philosophy and approach
- Current focus and learning goals
- Image-text card layout for visual storytelling

### 3. Projects Page (projects.html)
- Detailed project portfolio
- Project descriptions and technologies used
- Visual previews of work

### 4. Contact Page (contact.html)
- Comprehensive contact form with validation
- Form fields: name, email, phone, subject, message
- Contact information display list
- Response time expectations

## Key Features Implemented

### Semantic HTML
- Proper use of `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<figure>`
- Meaningful heading hierarchy (h1-h3)
- Descriptive landmarks for accessibility

### Forms & Validation
- Contact form with required field validation
- Proper label associations
- Multiple input types (text, email, tel, textarea)
- Organized fieldsets for better UX

### Accessibility Features
- Comprehensive alt text for all images
- Semantic HTML structure for screen readers
- Proper form labels and ARIA attributes
- Logical heading hierarchy
- Keyboard-navigable interface

### Content Elements
- Text formatting (paragraphs, lists)
- Definition list (`<dl>`) for contact information
- Unordered lists for navigation and skills
- Multiple link types (internal, external, mailto, tel)
- Images with proper width/height attributes and lazy loading

### Best Practices
- Consistent navigation across all pages
- Mobile-responsive viewport meta tag
- Proper DOCTYPE and language declaration
- Clean, organized, and indented code
- Meaningful class and ID naming conventions
